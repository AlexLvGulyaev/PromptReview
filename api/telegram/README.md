# Telegram Bot для Prompt Review Service

**UI-слой над FastAPI API для анализа промптов.**

---

## Архитектура

```
Telegram
    ↓
Telegram Bot (aiogram 3.x)
    ↓
POST /review
    ↓
FastAPI API (localhost:8000)
    ↓
Prompt Review Engine
    ↓
LangFlow / LangChain
```

**Ключевой принцип:** Бот не содержит бизнес-логику анализа. Вся логика остаётся в Prompt Review Service.

---

## Возможности

### Анализ промптов

Если текст является промптом, бот показывает:

- 📊 **Итоговую оценку качества** (excellent, good, fair, poor)
- 🎯 **Назначение промпта**
- 📈 **Оценки по критериям** (1–10):
  - Понятность
  - Полнота
  - Отсутствие неоднозначностей
  - Соответствие аудитории
  - Формат результата
  - Качество ограничений
  - Достаточность предположений
  - Структурированность
- ✅ **Сильные стороны**
- ⚠️ **Слабые стороны**
- 💡 **Рекомендации по улучшению** (с приоритетами)
- 📝 **Улучшенную редакцию промпта**

### Анализ обычного текста

Если текст не является промптом:

- 🔍 **Причина**, почему текст не распознан
- 🔄 **Варианты преобразования** в промпт

## Установка

### 1. Требования

- Python 3.10+
- FastAPI API запущен на `http://localhost:8000`
- Telegram Bot Token (получить у @BotFather)

### 2. Зависимости

```bash
cd api/telegram
pip install -r requirements.txt
```

### 3. Переменные окружения

Создайте файл `.env` в каталоге `api/` или установите переменные окружения:

```bash
# Telegram Bot Token (обязательно)
TELEGRAM_BOT_TOKEN=your_telegram_bot_token

# URL FastAPI API (по умолчанию http://localhost:8000)
PROMPT_REVIEW_API_URL=http://localhost:8000

# Таймаут API в секундах (по умолчанию 60)
API_TIMEOUT=60
```

Или используйте боевой `.env` из `infra/`:

```bash
cp ../../infra/.env .
```

## Запуск

### Локальный запуск (разработка)

```bash
# Убедитесь, что FastAPI API запущен
cd ../..  # корень проекта
uvicorn api.app.main:app --reload

# В другом терминале запустите бота
cd api/telegram
python bot.py
```

### Docker (production)

См. `infra/docker-compose.api.yml` для конфигурации контейнера.

## Форматирование

Бот использует **MarkdownV2** для форматирования сообщений:

- **Заголовки:** `*текст*` (bold)
- **Код:** ` ```код``` `
- **Списки:** `• пункт`
- **Эмодзи:** для визуального выделения

Функция экранирования спецсимволов: `escape_markdown_v2()`

## Обработка ошибок

Бот обрабатывает все типы ошибок API:

| Ошибка | Сообщение пользователю |
|--------|------------------------|
| `api_unavailable` | Сервис анализа временно недоступен. Попробуйте позже. |
| `timeout` | Превышено время ожидания ответа. Попробуйте отправить более короткий текст. |
| `invalid_response` | Получен некорректный ответ от сервера. |
| `network_error` | Ошибка сети. Проверьте подключение к интернету. |
| `unknown` | Произошла неизвестная ошибка. Попробуйте позже. |

Понятность: ████████░░ 8/10
Полнота: ██████░░░░ 6/10

• Чётко указана роль (опытный маркетолог)

## Тестирование

### E2E-тестирование

Выполните следующие сценарии:

1. **Команда /start**
   - Отправьте `/start` в Telegram
   - Ожидайте приветственное сообщение с описанием возможностей

2. **Анализ промпта**
   - Отправьте корректный промпт
   - Ожидайте полный анализ с оценками и рекомендациями

3. **Анализ обычного текста**
   - Отправьте обычный текст (не промпт)
   - Ожидайте объяснение и варианты преобразования

4. **Обработка ошибки API**
   - Остановите FastAPI API
   - Отправьте текст в бота
   - Ожидайте понятное сообщение об ошибке

### Скриншоты

После тестирования сохраните скриншоты:

- `/start` — приветственное сообщение
- Анализ промпта — результат анализа
- Анализ обычного текста — результат анализа
- Ошибка API — сообщение об ошибке

## Troubleshooting

### Бот не запускается

**Ошибка:** `TELEGRAM_BOT_TOKEN не указан`

**Решение:** Установите переменную окружения или создайте `.env` файл.

---

### Бот отвечает "Сервис анализа недоступен"

**Причина:** FastAPI API не запущен или недоступен.

**Решение:**
1. Проверьте, запущен ли FastAPI: `curl http://localhost:8000/health`
2. Проверьте `PROMPT_REVIEW_API_URL` в `.env`

---

### Таймаут при ответе

**Причина:** Backend (LangFlow/LangChain) долго обрабатывает запрос.

**Решение:** Увеличьте `API_TIMEOUT` в `.env`.
