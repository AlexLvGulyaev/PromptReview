## Исходное задание

Переработать `langflow/README.md` так, чтобы он описывал именно реализацию Prompt Review Agent в LangFlow из PEl03.

**Важно:**
- README должен документировать компонент LangFlow, а не архитектуру PEl05
- Основным источником истины — отчёт PEl03
- Не придумывать новые архитектурные решения

## Принятые решения

### Изменение фокуса документации

**Было:** Проектное описание будущей реализации для архитектуры n8n + LangFlow (PEl05)

**Стало:** Документация фактической реализации из учебного модуля PEl03

### Изученные материалы PEl03

**Ключевые факты из отчёта:**

1. **Основа:** Шаблон Basic Prompting из LangFlow
2. **Архитектура:** Базовая архитектура сохранена без изменений
3. **Изменения:** Только системный промпт
4. **Назначение:** Анализ качества промптов

**Поведение агента:**
- Проверка: является ли текст промптом
- Если нет — объяснение и варианты преобразования
- Если да — структурированное ревью
- Количественная оценка по критериям
- Рекомендуемая редакция с сохранением цели

**Тестирование:**
- Учебный промпт по Python
- Простой промпт с минимальными требованиями
- Объёмный инженерный промпт (пресейл-отклики)
- Обычный текст, не являющийся промптом

**Устранённые недостатки:**
- Анализ для не-промптов
- Отсутствие количественной оценки
- Неразделённые рекомендации
- Полное переписывание вместо редакции

**Направления развития (не реализованы):**
- Сохранение в БД
- История версий
- Сравнение редакций
- REST API
- Интеграция с n8n
- Административный интерфейс
