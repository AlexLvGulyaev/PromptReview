### Задачи

#### Задача №1. Построить карту эволюции проекта

Для каждого этапа (PEl03–PEl06) определить:

- назначение реализации;

- собственную архитектуру;
- какие артефакты были унаследованы от предыдущего этапа;
- какие были созданы заново.

Представить результат в виде последовательной схемы развития проекта.

#### Задача №2. Определить Source of Truth каждого этапа

Для каждого урока определить:

- где находится основной системный промпт;
- где определяется структура ответа;
- где определяется API;
- где определяется JSON-контракт;
- где определяется бизнес-логика анализа.

Если один и тот же артефакт существует в нескольких версиях — явно показать различия.

#### Задача №3. Локализовать замечания Антона

Для каждого замечания определить:

- К какому уроку оно относится.
- На какой артефакт оно направлено.
- Является ли этот артефакт:
  - локальным для данного урока;
  - переиспользуемым в последующих уроках;
  - частью общего ядра Prompt Review.
- Если замечание реализовать, определить предполагаемую область влияния:
  - только данный урок;
  - данный урок и последующие реализации;
  - весь проект Prompt Review.

Обязательно обосновать каждый вывод.

#### Задача №4. Определить каноническую реализацию Prompt Review

На основании текущего состояния проекта определить:

- Какая именно интеллектуальная реализация сегодня является основной для Prompt Review Service.
- Если в разных частях проекта используются разные реализации, подробно описать механизм их взаимодействия.
- Если такой механизм нигде явно не зафиксирован, отметить это как архитектурный пробел.

#### Задача №5. Подготовить рекомендации

Не выполнять никаких изменений.

Подготовить рекомендации в виде таблицы:

| Замечание | Уровень локализации | Требует изменения только урока | Требует изменения ядра проекта | Возможные последствия |

#### PEl05: n8n интеграции (Урок)

**Назначение:** Интеграция AI-компонента с платформой n8n, два интеграционных сценария.

**Технологии:**
- n8n (оркестратор)
- LangFlow (сценарий 1: HTTP Request → LangFlow API)
- LangChain Code Node (сценарий 2: весь интеллект внутри n8n)
- OpenAI API (LLM runtime)

**Собственный системный промпт:**
- ✅ Да, **модульный конвейер**
- **Сценарий 1 (LangFlow):** использует промпт из PEl03
- **Сценарий 2 (LangChain Code):** модульная декомпозиция:
  - `extractInput` — извлечение текста
  - `collectPromptMetrics` — инструментальный расчёт метрик
  - `classifyPrompt` — классификация (промпт/не промпт)
  - `reviewPrompt` — анализ качества
  - `rewritePrompt` — улучшенная редакция
  - `composeResult` / `composeNotPrompt` — формирование ответа

**Собственный JSON-контракт:**
- ✅ Да, **впервые введён**
- Структура `PromptReviewResponse` с полями:
  - `request_id`, `user_id`, `is_prompt`
  - `purpose`, `strengths`, `weaknesses`, `recommendations`
  - `scores` (8 критериев + overall)
  - `quality_level` (excellent, good, fair, poor, not_applicable)
  - `revised_prompt`
  - `reason`, `conversion_options` (для is_prompt=false)
  - `metrics`, `processing_time_ms`

**Собственная архитектура:**
- ✅ Да
- **Сценарий 1:** n8n Workflow → HTTP Request → LangFlow API → Structured Output → JSON
- **Сценарий 2:** Chat Trigger → LangChain Code → OpenAI Chat Model → JSON Output
- Ветвление по `is_prompt`: разные пути для промптов и обычных текстов

**Унаследовано от PEl03-PEl04:**
- Концепция Prompt Review Agent
- Критерии качества
- Инструмент метрик (из PEl04)
- Методология анализа

**Создано заново:**
