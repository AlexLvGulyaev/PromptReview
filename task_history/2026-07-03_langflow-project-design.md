## Принятое архитектурное решение

### Контекст

LangFlow-реализация переносит существующий Prompt Review Agent из учебного окружения PEl03 в инфраструктуру VPS для использования в качестве AI-компонента в архитектуре n8n + LangFlow.

### Изученные материалы PEl03

**Ключевые принципы из урока:**

1. **MVP-архитектура в LangFlow:**
   - Линейная цепочка: User Input → Prompt → LLM → Output
   - Чёткое разделение ответственности между компонентами
   - Управляемый процесс вместо одиночного вызова LLM

2. **Компоненты Flow:**
   - Chat Input — точка входа
   - Prompt Template — системные инструкции
   - Chat Model — языковая модель
   - Structured Output — структурированный результат
   - Chat Output — возврат ответа

3. **Интеграционный подход:**
   - LangFlow как AI-сервис, вызываемый через API
   - n8n как оркестратор бизнес-логики
   - REST API для взаимодействия

### Архитектура Prompt Review Agent

**Поток данных:**

```
Пользователь → n8n → HTTP Request → LangFlow API → Prompt Review Flow → Structured JSON → n8n → Ответ
```

**Компоненты Flow:**

- Chat Input — входной промпт
- Prompt — системный промпт для анализа качества
- Chat Model — Ollama с gemma4:e4b
- Structured Output — JSON-структура результата
- Chat Output — возврат структурированного ответа

### Structured Output

Проектируется сразу со структурированным выводом (в отличие от учебного варианта PEl03).

**Предварительная структура JSON:**
- purpose — назначение промпта
- strengths — сильные стороны
- weaknesses — недостатки
- recommendations — рекомендации
- scores — оценки по критериям
- overall_score — итоговая оценка
- improved_prompt — рекомендованная редакция

### Интеграция с n8n

**Принципы:**
- LangFlow — независимый AI-сервис
- n8n — оркестратор
- REST API — протокол взаимодействия
- JSON — формат данных

### Развёртывание

**Предполагаемый способ:**
- Docker-контейнер на VPS
- Отдельный поддомен
- HTTPS через Traefik
- REST API для вызова из n8n
