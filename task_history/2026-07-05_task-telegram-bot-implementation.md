# Задача: Реализация Telegram Bot для Prompt Review Service

## Исходное задание

Реализовать этап 6 IMPLEMENTATION_PLAN — Telegram Bot для Prompt Review Service.

### Требования:

1. **Архитектура**: Telegram Bot как самостоятельный UI-слой над существующим FastAPI API.
   - Бот не должен содержать бизнес-логику анализа
   - Вся логика остаётся в Prompt Review Service
   - Бот вызывает только endpoint `/review`

2. **Пользовательский сценарий**:
   - Команда `/start` — краткое описание возможностей, приглашение отправить текст
   - Пользователь отправляет произвольный текст
   - Бот вызывает `/review` API
   - Красивое отображение результата

3. **Форматирование результата для промпта**:
   - Итоговая оценка качества
   - Назначение
   - Сильные стороны
   - Недостатки
   - Рекомендации
   - Улучшенная редакция промпта

4. **Обработка не-промптов**:
   - Сообщение, что текст не распознан как промпт
   - Объяснение причины
   - Варианты преобразования в промпт

5. **Интерфейс**:
   - Читаемое форматирование (заголовки, эмодзи, списки)
   - MarkdownV2 или HTML (последовательно)
   - Не выводить "сырой" JSON

6. **Обработка ошибок**:
   - FastAPI недоступен
   - Timeout
   - Некорректный ответ
   - Ошибки сети

7. **Документация**:
   - `telegram/README.md`
   - Инструкции по запуску
   - Переменные окружения
   - Примеры использования

8. **E2E-тестирование**:
   - `/start`
   - Анализ корректного промпта
   - Анализ обычного текста
   - Обработка ошибки API
   - Скриншоты всех сценариев

9. **Обновление артефактов**:
   - `PROJECT_STATE.md`
   - `README.md`
   - `task_history`

**Решение:**
- HTML-форматирование (parse_mode="HTML")
- Экранирование через `html.escape()` для текстовых данных
- Теги: `<b>`, `<i>`, `<pre>`

## Подробный статус тестирования

### Проблемы, обнаруженные при тестировании

#### Проблема 1: Конфликт polling в Telegram

**Симптом:**
```
Conflict: terminated by other getUpdates request
```

**Причина:**
- Уже запущен другой процесс бота с тем же токеном (`python run_telegram_bot.py`, PID 1126701)
- Telegram разрешает только одно polling-соединение для одного токена

**Решение:**
```bash
kill -9 1126701  # Остановлен конфликтующий процесс
```

---

#### Проблема 2: Ошибка MarkdownV2

**Симптом:**
```
Bad Request: can't parse entities: Character '.' is reserved and must be escaped with the preceding '\'
```

**Причина:**
- MarkdownV2 требует экранирования спецсимволов: `_ * [ ] ( ) ~ \` > # + - = | { } . !`
- Числа с точкой (например, `overall: 7.2`) создавали неэкранированные точки
- Экранирование применялось неправильно: разметка и данные смешивались

**Попытки решения:**

1. **Первая попытка:** Экранировать все спецсимволы в функции `escape_markdown_v2()`
   - Не сработало: кеширование Python, старая версия загружалась

2. **Вторая попытка:** Очистка кеша `__pycache__` и перезапуск бота
   - Не сработало: экранирование точки в числе `7.2` всё ещё вызывало ошибку

3. **Финальное решение:** Переключение на HTML-форматирование

**Реализация HTML:**

```python
# bot.py
await message.answer(
    formatted_response,
    parse_mode="HTML",  # Вместо "MarkdownV2"
    disable_web_page_preview=True
)

# formatter.py
from html import escape as html_escape

# Экранирование только текстовых данных
sections.append(f"• {html_escape(strength)}")

# HTML-теги для форматирования
sections.append(f"<b>{emoji} Анализ промпта</b>")
```

**Результат:**
- ✅ Точки в числах (`7.2`) не требуют экранирования в HTML
- ✅ `html.escape()` автоматически экранирует `<`, `>`, `&`
- ✅ Все сообщения корректно отображаются
